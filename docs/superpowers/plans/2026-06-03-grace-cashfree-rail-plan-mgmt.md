# Grace Cashfree rail + plan-management ‚Äî Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Extend Grace (the codegen tool) so `grace generate cashfree --domain subscriptions` reproduces the lens 0.3.0 customer-chosen-rail + 0.4.0 plan-management hand-edits, then run one combined regen to parity ‚Äî with zero hand-edits to marker-stamped files.

**Architecture:** Grace bundles a rulebook (markdown patterns + mirror docs) + per-PSP `connector_docs` + a built prompt, and invokes a Claude Code backend that writes the lens connector. We change Grace's *inputs* (rulebook patterns, mirror docs) **and** its *pipeline code* (`prompt.py` locked-field/import blocks, `quality_rubric.py` method set, `context.py` rulebook registration) so the new shape is permitted, instructed, and gated. Grace's own pytest suite pins all of this via content-assertion tests (`tests/test_rulebook_shape.py`, `tests/pipeline/test_prompt_domain.py`, `tests/pipeline/test_context_domain.py`, `tests/test_quality_rubric_v2.py`), so almost every task is real TDD. The final two phases bump the version and run the combined regen, verifying behavioral parity against the committed hand-edited lens tree.

**Tech Stack:** Python 3.11+, pytest, mypy --strict, black -l 100. Two repos: **grace** = `/Users/sarthak/PycharmProjects/references/grace` (branch `mandates`); **lens** = `/Users/sarthak/PycharmProjects/symplora/sylibs/packages/lens` (branch `feat/lens`).

**Source of truth:** the spec at `docs/superpowers/specs/2026-06-03-grace-cashfree-rail-plan-mgmt-spec.md`. Read it before starting. This plan implements that spec's ¬ß6 requirements (A1‚ÄìC2, X1‚ÄìX5) and ¬ß10 sequencing.

```python
async def create_plan(self, request: CreatePlanRequest) -> CreatePlanResponse:
    # Deterministic plan id: caller value, else derive from the idempotency key so an
    # idempotent retry sends an identical body. (Charset: alnum/dot/hyphen/underscore, <= 40.)
    plan_id = request.merchant_plan_id or request.idempotency_key
    psp_req = <Psp>CreatePlanRequest(
        plan_id=plan_id,
        plan_name=plan_id,                       # lens has no separate name
        plan_type="PERIODIC",                    # CHANGE_PLAN is PERIODIC-only
        plan_recurring_amount=request.recurring_amount.minor_units / 100,   # major units
        plan_max_amount=request.max_amount.minor_units / 100,
        plan_currency=request.recurring_amount.currency.value,
        plan_intervals=request.interval_count,
        plan_interval_type=request.interval_type.value,
    )
    headers = build_auth_headers(self._config, psp_req)
    headers["<psp-idempotency-header>"] = request.idempotency_key
    try:
        resp = await self._client.post("/plans", json=psp_req.model_dump(exclude_none=True), headers=headers)
        resp.raise_for_status()
    except httpx.HTTPStatusError as e:
        raise _map_http_error(e) from e          # 400 ‚Üí INVALID_REQUEST, etc.
    except httpx.HTTPError as e:
        raise ConnectorError(reason=ConnectorErrorReason.PSP_UNAVAILABLE) from e
    try:
        psp_resp = <Psp>PlanEntity.model_validate(resp.json())
    except ValidationError as e:
        raise ConnectorError(reason=ConnectorErrorReason.INTERNAL, psp_message=str(e)) from e
    return CreatePlanResponse(plan_id=str(psp_resp.plan_id or plan_id), raw=psp_resp.model_dump())
```
